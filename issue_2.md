# Issue 2: Googleアカウントでの認証機能の実装

## 概要
Supabase Authを利用して、Googleアカウントを使用したユーザー認証（SSO）を実装する。これにより、アプリケーションの利用を認証済みユーザーに限定し、ユーザーごとの履歴管理の基盤を作る。

## 詳細タスク

### 1. Supabaseプロジェクト設定
- [ ] SupabaseダッシュボードでGoogleプロバイダーを有効化する。
- [ ] Google Cloud ConsoleでOAuth同意画面とクレデンシャルを作成し、Client IDとSecretをSupabaseに設定する。

### 2. フロントエンド実装 (`web/`)
- [ ] `web/` ディレクトリ（Next.js）にてSupabaseクライアントのセットアップを行う（`@supabase/auth-helpers-nextjs` 等を利用）。
- [ ] **ログインページ**: Googleでログインボタンを配置したログイン画面を実装する。
- [ ] **認証ガード**: 未認証ユーザーがトップページや推論機能にアクセスした場合、ログインページへリダイレクトさせる処理（MiddlewareまたはLayoutでの制御）を実装する。
- [ ] **ユーザー情報取得**: ログイン成功後、ヘッダー等にユーザーのアイコンや名前を表示する。

## 完了条件
- ユーザーが「Googleでログイン」ボタンをクリックし、Googleアカウントでログインできること。
- ログイン状態でなければ、画像アップロード画面にアクセスできないこと。
- ログアウト機能が動作すること。
